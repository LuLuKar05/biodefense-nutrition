# Biodefense Nutrition Skill

You are a biodefense nutrition assistant operating in **message router mode**. OpenClaw forwards ALL user messages to the Python Agent Orchestrator via the Gateway Bridge. The Python backend handles everything — you just relay messages and replies.

## Architecture — How It Works

```
User (any channel)
    │
    ▼
OpenClaw Gateway (:18789) ── multi-channel connector
    │ POST /hooks/agent
    ▼
Gateway Bridge (:18790) ── Python FastAPI
    │ calls orchestrator.route_message()
    ▼
Agent Orchestrator ── routes to:
    ├── Onboarding Agent (FLock LLM)
    ├── Nutrition Agent (FLock LLM + local tools)
    └── Threat Handler → Layer 3 Backend (:8100)
```

## Privacy Rules (CRITICAL)

1. **NEVER send user health data (name, age, weight, allergies, diet, goals) to any external API.**
2. User profile data is stored ONLY in local JSON files (`data/profiles/<user_id>.json`).
3. The ONLY outbound calls:
   - User messages → FLock API (`api.flock.io/v1`) for LLM inference
   - City name → Layer 3 Threat API (anonymous, no user identity)
4. Meal plans are generated by the Python orchestrator using FLock LLM + local tools.
5. TDEE/macro calculations run locally via `agents/tools/macro_calculator.py`.

## Three-Layer Architecture

### Layer 1: OpenClaw Gateway (this)
- Multi-channel connector (Telegram, Discord, WhatsApp, Slack, WebChat, 20+)
- Forwards ALL messages to the Python orchestrator via Gateway Bridge
- Receives proactive alerts and delivers to users

### Layer 2: Agent Orchestrator (Python)
- Per-user personal agent — routes intent to specialized handlers
- **Onboarding Agent** — FLock-powered natural language profile collection (9 fields)
- **Nutrition Agent** — meal planning, macro calculation, meal logging via FLock + local tools
- **Threat Handler** — fetches pre-formatted reports from Layer 3
- **Webhook Receiver** — receives and distributes proactive threat alerts
- **Auto-Subscribe** — when profile completes, subscribes user's city to Layer 3

### Layer 3: Threat Intelligence Backend (FastAPI :8100)
- **WHO DON API** — real outbreak data via OData (public, no auth)
- **NCBI Entrez** — protein sequence fetching for detected pathogens
- **Amina AI** — local amino acid analysis + 8-factor compound scoring
- **Research Agent** — FLock LLM for unknown diseases
- **Hybrid Nutrient Mapper** — 3-tier mapping (DB → Amina AI → LLM research)
- **Subscriber Registry** — fires webhooks to Layer 2 when threats change
- **Zero-Knowledge** — never receives or stores user PII

## Phase 1: Onboarding & Meal Planning (ALL LOCAL)

### Onboarding Flow
The Python Onboarding Agent uses FLock LLM to naturally extract 9 profile fields:
- name, age, sex, weight, height, activity_level, goal, diet, allergies, city
- All stored locally in `data/profiles/<chat_id>.json`
- On profile completion → auto-subscribe to Layer 3 for user's city

### Macro Calculation (LOCAL)
Uses `agents/tools/macro_calculator.py`:
- Mifflin-St Jeor TDEE with activity multiplier
- Goal-adjusted macros (cut/bulk/maintain)
- Returns `{ "tdee": 2200, "protein_g": 176, "carbs_g": 248, "fat_g": 73 }`

### Meal Plan Generation (LOCAL)
Uses `agents/tools/meal_planner.py` + FLock LLM:
- Respects diet type, allergies, macro targets
- Formatted for mobile: 🌅 Breakfast, 🌞 Lunch, 🌙 Dinner, 🍎 Snacks

### Meal Logging (LOCAL)
Uses `agents/tools/meal_manager.py`:
- Natural language logging via FLock
- Local JSON storage, macro estimation
- Daily balance tracking

## Phase 2: Threat Detection (WHO DON + NCBI)

### Real Outbreak Data
Layer 3 fetches from WHO Disease Outbreak News API:
- `https://www.who.int/api/news/diseaseoutbreaknews` (OData, public, no auth)
- 3-tier region filtering: city-level → country-level → global
- Real outbreaks with dates, disease names, affected regions

### Protein Sequences
NCBI Entrez API fetches real protein sequences for detected pathogens:
- Supports both known diseases (mapped to search terms) and novel ones
- Returns protein ID, title, organism, full amino acid sequence

## Phase 3-4: Amina AI Analysis (LOCAL)

### Protein Composition Analysis
Amina AI (`threat_backend/amina_ai.py`) runs LOCALLY:
- Amino acid frequency analysis of pathogen proteins
- 15 pathogen motif patterns (fusion peptides, receptor binding, proteolytic sites, etc.)
- Identifies structural signatures indicating virulence

### Compound Scoring
8-factor scoring of 15 natural food compounds against pathogen proteins:
- Amino acid composition affinity
- Motif interaction potential
- Molecular weight compatibility
- Hydrophobicity matching
- Net charge interaction
- Known antiviral activity
- Structural diversity bonus
- Bioavailability factor

Returns ranked food compounds with confidence scores, food sources, and mechanisms.

## Phase 5: Proactive Alerts (Webhook Push)

### How Proactive Alerts Work
1. Layer 3 runs background pipeline hourly (configurable)
2. When threats CHANGE for a city (new outbreaks, resolved outbreaks, new analysis):
   - Computes fingerprint hash of current threat state
   - Detects change from previous fingerprint
   - Fires webhooks to all subscribers for that city
3. Gateway Bridge receives the alert
4. Looks up which users live in that city
5. Pushes alert to each user via OpenClaw Gateway → their channel
6. Auto-chains meal adaptation: generates new meal plan featuring biodefense foods

### Alert Content
- Pre-formatted rich report with threat summary, protein analysis, compound scores
- Priority food recommendations (top compounds mapped to everyday foods)
- Automatically adapted meal plan from Nutrition Agent
